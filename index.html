Theory of CFG Simplification

To safely minimize a Context-Free Grammar without changing the language it generates, we apply systematic algorithms in a specific pipeline order. Explore the theory below before trying it yourself.

1

Useless Symbols

We eliminate variables that don't produce outcomes.

Phase 1: Non-Productive. Remove variables that never derive a string consisting only of terminals.
Phase 2: Unreachable. From the Start symbol (S), build a reachable graph and remove anything disconnected.

2

Null Productions

We remove productions that derive the empty string (? or ε).

First, we identify "nullable" variables. Then, for any rule containing these variables, we generate combinations of sub-rules substituting those variables with the empty string, allowing us to safely omit A → ?.

3

Unit Productions

We streamline the grammar by collapsing direct variable-to-variable chains (e.g., A → B).

We calculate a "derivation closure" for every variable, and directly substitute B's non-unit derivations into A, completely removing the intermediate step.

Stage 1: Useless Symbols (Initial Sweep)

Eliminates non-productive symbols and symbols unreachable from the start state.

Algorithm: Remove Useless Symbols 
1. Compute Generating Set (G) [Productive Symbols]:
   - Add all variables that directly derive strings of terminals.
   - Iteratively add variables that derive strings of symbols currently in G.
   - Remove any rules containing symbols not in G.
2. Compute Reachable Set (R):
   - Start with R = {S}.
   - Iteratively add any variable that appears on RHS of rules mapped from variables in R.
   - Remove any rules for variables not in R.

Stage 2: Null Productions Removal

Identifies nullable non-terminals (deriving ?) and dynamically generates alternative non-null productions.

Algorithm: Remove ε-Productions 
1. Compute Nullable Set (N):
   - Add variables that directly derive ε.
   - Iteratively add variables that derive strings entirely composed of symbols in N.
2. For each rule containing symbols in N:
   - Generate combinations replacing each occurrence with ε.
   - Add newly generated rules explicitly.
   - Remove all ε-productions from the grammar.

Stage 3: Unit Productions Removal

Eliminates A → B transitions by determining derivation closures and making direct substitutions.

Algorithm: Remove Unit Productions 
1. Compute Derivation Closure D(A) for every variable:
   - Initialize D(A) = {A}.
   - If B is in D(A) and there is a unit rule B → C, add C to D(A).
2. For each variable A:
   - For every variable B in D(A), map all non-unit rules of B to A.
   - Remove the original unit rules (A → B).

Algorithm: Final Sweep
